Nginx PHP 8.4 MariaDB Git
PASSO 00

Preparar o sistema

Antes de qualquer coisa, atualize os pacotes do sistema para garantir que tudo esteja na versão mais recente.

bash 
# Atualizar lista de pacotes e instalar atualizações
sudo apt update && sudo apt upgrade -y

# Instalar utilitários básicos
sudo apt install -y curl wget unzip git
PASSO 01

Instalar Nginx

O Nginx é o servidor web que vai receber as requisições e repassar ao PHP.

bash 
# Instalar o Nginx
sudo apt install -y nginx

# Habilitar e iniciar o serviço automaticamente
sudo systemctl enable --now nginx

# Verificar se está rodando (deve mostrar "active (running)")
sudo systemctl status nginx
✓

Teste: Acesse http://SEU-IP no navegador — deve aparecer a página de boas-vindas do Nginx.

bash 
# Remover o virtual host padrão para evitar conflitos
sudo rm /etc/nginx/sites-enabled/default
PASSO 02

Instalar PHP 8.4

O PHP-FPM processa os scripts PHP e se comunica com o Nginx via socket Unix.

bash 
# Instalar PHP-FPM e extensões necessárias
sudo apt install -y php-fpm php-mysql php-curl php-mbstring php-json

# Habilitar e iniciar o PHP-FPM
sudo systemctl enable --now php8.4-fpm

# Confirmar a versão instalada
php --version
⚠

No Debian 13 a versão padrão é 8.4. No Ubuntu 24.04 pode ser 8.3. Se necessário, substitua php8.4-fpm pela versão instalada. Verifique com ls /run/php/.

PASSO 03

Instalar MySQL / MariaDB

O MariaDB é compatível com MySQL e é o banco de dados padrão no Debian moderno.

bash 
# Instalar MariaDB
sudo apt install -y mariadb-server

# Habilitar e iniciar
sudo systemctl enable --now mariadb

# Executar o assistente de segurança
sudo mariadb-secure-installation

Respostas recomendadas no assistente

Enter current password for root ↵ Enter (sem senha ainda)
Switch to unix_socket authentication? N
Change the root password? Y → defina uma senha forte
Remove anonymous users? Y
Disallow root login remotely? Y
Remove test database? Y
Reload privilege tables? Y
PASSO 04

Instalar Git

O Git permite versionar o código e clonar repositórios diretamente no servidor.

bash 
# Instalar Git
sudo apt install -y git

# Verificar a instalação
git --version

# Configurar identidade (opcional, para commits)
git config --global user.name "Seu Nome"
git config --global user.email "seu@email.com"
PASSO 05

Criar o banco de dados

Execute o script SQL para criar o banco, tabelas e dados de exemplo de uma só vez.

Opção A — Arquivo SQL (recomendado)

bash 
# Importar o script database.sql diretamente
sudo mysql -u root -p < /var/www/crud-produtos/database.sql

Opção B — Manualmente no shell

bash 
sudo mysql -u root -p
sql 
-- Criar banco de dados
CREATE DATABASE IF NOT EXISTS crud_produtos
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

USE crud_produtos;

-- Tabela de categorias
CREATE TABLE IF NOT EXISTS categorias (
  id          INT AUTO_INCREMENT PRIMARY KEY,
  nome        VARCHAR(100) NOT NULL,
  criado_em   TIMESTAMP DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB;

-- Tabela de produtos
CREATE TABLE IF NOT EXISTS produtos (
  id           INT AUTO_INCREMENT PRIMARY KEY,
  nome         VARCHAR(200) NOT NULL,
  descricao    TEXT,
  preco        DECIMAL(10, 2) NOT NULL DEFAULT 0.00,
  estoque      INT NOT NULL DEFAULT 0,
  categoria_id INT,
  imagem_url   VARCHAR(500),
  ativo        TINYINT(1) NOT NULL DEFAULT 1,
  criado_em    TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  atualizado_em TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (categoria_id) REFERENCES categorias(id) ON DELETE SET NULL
) ENGINE=InnoDB;

-- Dados de exemplo

  ('Eletrônicos'), ('Roupas'), ('Alimentos');

INSERT INTO produtos (nome, descricao, preco, estoque, categoria_id) VALUES

  ('Notebook Pro', 'Notebook i5, 8GB RAM, SSD 256GB', 2899.99, 8, 1),
  ('Camiseta Algodão', '100% algodão', 49.90, 100, 2);

-- Criar usuário para a aplicação
CREATE USER IF NOT EXISTS 'crud_user'@'localhost'
  IDENTIFIED BY 'SenhaForte@2024';

PASSO 06

Implantar os arquivos do site

1 — Criar o diretório

bash 
sudo mkdir -p /var/www/crud-produtos/{api,config}

2 — Enviar os arquivos

Opção A — SCP do seu computador local:

bash 
scp -r ./crud-produtos/* usuario@SEU-IP:/var/www/crud-produtos/

Opção B — Clonar do Git:

bash 
cd /var/www
sudo git clone https://github.com/SEU-USUARIO/crud-produtos.git

3 — Ajustar permissões

bash 
# Dar posse ao usuário www-data (usuário do Nginx)
sudo chown -R www-data:www-data /var/www/crud-produtos
sudo chmod -R 755 /var/www/crud-produtos

4 — Editar credenciais do banco

bash 
sudo nano /var/www/crud-produtos/config/database.php
php 
// Ajuste estes valores conforme o que você definiu no Passo 05
define('DB_USER', 'crud_user');
define('DB_PASS', 'SenhaForte@2024'); // ← mesma senha do passo 5
PASSO 07

Configurar o Nginx

1 — Criar o virtual host

bash 
sudo nano /etc/nginx/sites-available/crud-produtos

Cole o conteúdo abaixo no arquivo:

nginx 
server {
    listen 80 default_server;
    listen [::]:80 default_server;

    root /var/www/crud-produtos;
    index index.html index.php;
    server_name _;
    charset utf-8;
    client_max_body_size 10M;

    location / {
        try_files $uri $uri/ =404;
    }

    # Processar PHP via FastCGI
    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
        fastcgi_pass unix:/run/php/php8.4-fpm.sock;
    }

    # Bloquear arquivos ocultos
    location ~ /\. {
        deny all;
    }

    gzip on;
    gzip_types text/plain text/css application/javascript application/json;

    access_log /var/log/nginx/crud-access.log;
    error_log  /var/log/nginx/crud-error.log;
}

2 — Ativar e reiniciar

bash 
# Criar symlink para ativar o site
sudo ln -s /etc/nginx/sites-available/crud-produtos \
           /etc/nginx/sites-enabled/

# Testar a sintaxe da configuração
sudo nginx -t

# Se OK, reiniciar o Nginx
sudo systemctl restart nginx
✓

Se o nginx -t retornar "syntax is ok / test is successful", a configuração está correta.

Endpoints da API

MétodoURLAção

POST /api/produtos.php Criar novo produto
PUT /api/produtos.php?id=1 Atualizar produto

Solução de problemas

Erro 502 Bad Gateway

Geralmente causado pelo socket do PHP-FPM incorreto.

bash 
# Verificar se o PHP-FPM está rodando
sudo systemctl status php8.4-fpm

# Ver qual socket está disponível
ls /run/php/
# Se aparecer php8.3-fpm.sock, ajuste no nginx.conf

Erro de conexão com banco de dados

Credenciais incorretas ou usuário não criado.

bash 
# Testar a conexão manualmente
mysql -u crud_user -p crud_produtos

# Recriar o usuário se necessário
sudo mysql -e "DROP USER IF EXISTS 'crud_user'@'localhost';"
sudo mysql -e "CREATE USER 'crud_user'@'localhost' IDENTIFIED BY 'SenhaForte@2024';"
sudo mysql -e "GRANT ALL PRIVILEGES ON crud_produtos.* TO 'crud_user'@'localhost';"
sudo mysql -e "FLUSH PRIVILEGES;"

Erro de permissão (403 Forbidden)

O Nginx não tem acesso aos arquivos.

bash 
sudo chown -R www-data:www-data /var/www/crud-produtos
sudo chmod -R 755 /var/www/crud-produtos
